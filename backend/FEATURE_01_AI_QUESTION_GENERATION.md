# FEATURE_01_AI_QUESTION_GENERATION.md

## Feature Name

AI Question Generation

---

## Purpose

Generate intelligent follow-up questions after idea analysis.

The goal is to identify missing information required to generate a complete project brief.

Questions should be generated by Gemini based on the submitted idea and the current brief state.

---

## Current State

Completed:

* Gemini Integration
* AI Service
* Prompt System
* Zod Validation
* Idea Analysis Endpoint

Current API:

POST /api/ai/analyze/:ideaId

This endpoint successfully returns:

* is_complete
* missing_fields
* questions

The response is currently returned directly to the client.

Questions are not persisted.

---

## Problem

Questions generated by AI are currently temporary.

The system has no way to:

* store generated questions
* track question status
* track answered questions
* reuse previous question generations

---

## Goal

Implement a complete Question Generation workflow.

## Implementation Report

### Architecture Decision
Questions are stored directly within the **Brief** model in an array. This avoids the overhead of a separate collection while maintaining a clean 1:1 relationship between a Brief and its refinement questions. It also simplifies ownership validation.

### Schema Changes
Modified `Brief` model:
*   Added `questions` field: Array of objects `{ key, question, status, createdAt }`.
*   `status`: Enum `['pending', 'answered']`.

### Routes
*   `POST /api/ai/analyze/:ideaId`: Now persists generated questions to the Brief.
*   `POST /api/ai/answers/:ideaId`: Updates question status to `answered` upon submission.

### Implementation Summary
1.  **Model**: Updated `brief.model.js` with the `questions` array.
2.  **Service**: Enhanced `ai.service.js` with lazy-loading for Gemini, structured JSON parsing, and Zod validation.
3.  **Controller**: 
    *   `analyzeIdea`: Now saves AI-generated questions and missing fields.
    *   `submitAnswers`: Now tracks question status when answers are provided.
4.  **Security**: Reused existing ownership validation via `getIdeaAndBrief` helper.
5.  **Robustness**: Handled Gemini markdown output by extracting JSON using regex.

**Status**: Completed.
